# RPC Mode

This document provides a detailed overview of the RPC Mode for the `oh-my-pi` coding agent, covering its protocol, usage, and integration points. RPC Mode enables programmatic control of the agent through a JSON-based stdin/stdout interface, making it suitable for embedding the agent in other applications .

## RPC Mode Overview

RPC Mode operates as a headless service, accepting `RpcCommand` objects as JSON lines on standard input and emitting `RpcResponse` objects and `AgentEvent` streams as JSON lines on standard output . This design facilitates integration with external applications .

### Protocol Overview

The RPC protocol consists of three main components:
*   **Commands**: JSON objects sent via stdin with a `type` field and an optional `id` for correlating requests and responses .
*   **Responses**: JSON objects emitted via stdout with `type: "response"`, the original `command` type, a `success` boolean, and optional `data` or `error` fields .
*   **Events**: `AgentSessionEvent` objects streamed as they occur, providing real-time updates on agent activity .

### Invocation

To run the agent in RPC mode, use the `--mode rpc` CLI argument .
```bash
omp --mode rpc [options]
```


### RPC Protocol Flow

The interaction flow in RPC mode is as follows:
```mermaid
sequenceDiagram
    participant Client
    participant stdin
    participant runRpcMode
    participant AgentSession
    participant stdout
    
    Client->>stdin: {"type":"ready"}
    runRpcMode->>stdout: ready signal
    
    Client->>stdin: {"id":"1", "type":"prompt", "message":"hello"}
    runRpcMode->>AgentSession: session.prompt("hello")
    runRpcMode->>stdout: {"id":"1", "type":"response", "success":true}
    
    loop Streaming
        AgentSession->>stdout: {"type":"agent_start"}
        AgentSession->>stdout: {"type":"message_start"}
        AgentSession->>stdout: {"type":"message_update", "delta":"..."}
        AgentSession->>stdout: {"type":"message_end"}
        AgentSession->>stdout: {"type":"agent_end"}
    end
    
    Client->>stdin: {"id":"2", "type":"get_state"}
    runRpcMode->>stdout: {"id":"2", "type":"response", "success":true, "data":{...}}
```


The `runRpcMode` function in `packages/coding-agent/src/modes/rpc/rpc-mode.ts` is responsible for handling incoming commands and dispatching them to the `AgentSession` . It also subscribes to `AgentSession` events and outputs them as JSON lines to stdout .
